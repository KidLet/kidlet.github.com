JavaScript学习笔记 （1）语法结构

语法结构是 编程语言中 最基础的语法，它定义了 应该怎样编写程序，变量名字的要求，注释的界定符号等等。

字符集

JavaScript程序编写采用的是 Unicode字符集。

Unicode 换码规则

因为有些系统无法输入所有Unicode字符，所以定义了一种换码规则，在 JavaScript的文本表示，或者正则表达式中，以 \u 开头后面跟着 4位 16进制码 来表示任意的 16-bit Unicode码，下面是一个例子

"café" === "caf\u00e9" // => true

Unicode的常态 (Normalization)

Unicode允许不同的编码 来表示同样的字符 比如 字符 “é”， 它既可以通过 \u00E9 也可以 用 \u0301 来表示，但是它在文本编辑器上看并没有什么不同。但是对JavaScript来说，“\u00E9”和“\u0301” 被认为是不同的。

区分大小写

JavaScript是 区分大小写 的语言。也就是说，关键字，变量名，函数名和其他标志符等等大小写是被认为不同的。

值得注意的是，在HTML中，是不区分大小写的（XHTML则除外），但是在JavaScript中这些必须被写成小写的。

比如在HTML 的 onclick 事件,有时候会被写成 onClick

但是在JavaScript中必须写为 onclick 才能生效

注释

注释是用来对代码进行解释说明，并且会被编译器（解释器）所忽略的部分。JavaScript 有两种风格的注释，并且和 C语言 的一样。

// 这是一行注释
以 // 开头到行的末尾 构成注释

/*
这也是注释
可以是多行
*/

以 /* 开始 ， 以 */ 结束的内容会被当成注释
注意， 不可以嵌套。

标识符和保留字

标识符

标识符：一个简单名字，用来给变量、函数等命名。 开头必须是，下划线（_），$符号, 英文字母 剩下的部分 可以是 下划线（_），$符号, 英文字母， 数字

保留字

JavaScript自己保留了许多标识符作为它自己的 关键字，所以不能将这些保留字作为 自己程序的标识符。 现在并不需要去背诵记住， 但是要有个大概的了解。

break   delete  function   return   typeof
case    do      if         switch   var
catch   else    in         this     void
continue    false     instanceof    throw   while
debugger    finally   new           true    with
default     for       null          try

并且 JavaScript 还预先定义了许多全局变量和函数，在进行编写代码的时候应该尽量避开。

arguments, Array, Boolean, Date, decodeURI, decodeURIComponent, encodeURI, encodeURIComponent, Error, eval, EvalError, Function, Infinity, isFinite, isNaN, JSON, Math, NaN, Number, Object, parseFloat, parseInt, RangeError, ReferenceError, RegExp, String, SyntaxError, TypeError, undefined, URIError

可选的分号

如果学过C语言， 一定会对分号“;”的作用非常了解。它通常被用来说明语句的结束。在JavaScript中， 可以省略分号，前提是在不引起歧义的情况下。但是我个人推荐不省略分号，因为那是一个好习惯， 它会让你的代码看起来更整洁。

小结

从本节中， 可以发现JavaScript的很多特性与C语言保持一致。
